## Implementation Workflow

### Before Implementation
Gather context to ensure successful task planning:

| Source | Gather |
|--------|--------|
| **Project Requirements** | Scope, objectives, deliverables, constraints |
| **Resource Availability** | Team capacity, skills, time commitments |
| **Timeline Constraints** | Deadlines, milestones, external dependencies |
| **Risk Factors** | Potential obstacles, uncertainties, mitigation needs |

### Step 1: Scope Definition
1. Clarify project objectives and deliverables
2. Identify stakeholders and their requirements
3. Define success criteria
4. Establish boundaries and constraints

### Step 2: Task Breakdown
1. Decompose project into major components
2. Further break down into manageable tasks
3. Identify task dependencies
4. Estimate effort and duration

### Step 3: Prioritization
1. Apply prioritization framework
2. Consider business value and urgency
3. Account for dependencies
4. Balance workload

### Step 4: Schedule Creation
1. Sequence tasks appropriately
2. Allocate resources effectively
3. Build in buffer time
4. Set realistic milestones

### Step 5: Tracking Setup
1. Define progress metrics
2. Establish reporting cadence
3. Set up monitoring tools
4. Plan for adjustments

## Quality Assurance

### Completeness Checks
- Are all deliverables accounted for?
- Are dependencies properly identified?
- Are resources adequately allocated?
- Are risks addressed?

### Realism Checks
- Are timelines achievable?
- Are resources sufficient?
- Are buffers appropriate?
- Are milestones meaningful?

## Troubleshooting

### Overwhelmed by Tasks
- Focus on high-priority items
- Break large tasks further
- Delegate when possible
- Eliminate non-essential tasks

### Missing Deadlines
- Reassess time estimates
- Identify bottleneck tasks
- Reallocate resources
- Adjust scope if necessary

### Scope Creep
- Revisit original requirements
- Assess impact of additions
- Update timeline and resources
- Communicate changes
